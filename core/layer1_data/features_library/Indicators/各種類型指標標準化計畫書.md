指標標準化與特徵硬化終極計畫書 (SOTA 250+ 最終版)

0. 核心宗旨：Alpha 零損耗轉譯

本計畫旨在將原始指標（Raw Indicators）轉譯為具備「空間感知」與「數值韌性」的高維特徵。修改後的指標必須確保：

尺度無關性 (Scale Invariance)：價格在 1 萬點與 10 萬點時，特徵分佈必須一致。

數學穩定性 (Numerical Stability)：徹底根除 NaN, Inf 與「協方差矩陣非正定」錯誤。

路徑依賴性 (Path Dependency)：保留類似 MC 策略中 BarsSinceEntry 的動態記憶能力。

1. 四大受管制數據類型：轉化契約

【類別 A】絕對價格類 (Absolute Price)

對象： VWAP, SMA, EMA, BB, POC/VAH/VAL。

嚴禁： 禁止直接輸出任何價格數值。

轉化公式： feat_X_dist = (Close / Indicator) - 1

物理意義： 將「價格空間」映射至「邏輯偏移百分比」。

【類別 B】累積/非平穩類 (Cumulative / Non-stationary)

對象： OBV, ADL, CMF, 累積成交量。

嚴禁： 禁止輸出隨時間無限增長/減少的序列。

轉化公式： feat_X_roc = (Indicator - EMA(Indicator, n)) / ATR(n)

物理意義： 將累積總量轉化為相對於波動率的「動態噴發強度」。

【類別 C】稀疏/布林類 (Sparse / Boolean)

對象： Liquidity Sweep (EQH/EQL), SMC OrderBlocks, FVG 觸發。

嚴禁： 禁止輸出長時間為 0 的死線（會導致矩陣奇異）。

轉化公式 (衰減機轉)：

觸發 T = 1.0 (或 -1.0)

隨後 T+n = Value[T+n-1] \* Decay_Factor (推薦 0.9 或從參數傳入)

物理意義： 將「瞬時信號」轉化為「具備記憶衰減的場域特徵」。

【類別 D】離散狀態類 (Discrete States)

對象： Force_Context (State), 趨勢分類 (-1, 0, 1)。

修正規範： 增加「邏輯滿足度」輸出。

實作： 與其只回傳 1，必須同時回傳 (Current_Val / Threshold)。

物理意義： 提供模型「接近觸發點」的漸進式資訊。

2. 統一輸出：三位一體協議 (Triple-Threat Protocol)

所有指標必須嘗試輸出以下「黃金組合」欄位，確保 AI 能看見「趨勢的呼吸」與「空間記憶」：

欄位後綴

名稱

數學邏輯

目的

\_pos

位置項

(Current / Reference) - 1

解決尺度與平移問題。

\_osc

動態項

\_pos - EMA(\_pos, 5)

創造 MACD 式的起伏畫面，辨識背離。

\_mem

記憶項

Close / Highest(Close, BarsSinceLastEvent)

模擬 BarsSinceEntry，繞過橫盤，定位波峰距離。

註：BarsSinceLastEvent 中的「事件」由指標開發者定義（如：交叉、擺動點形成、政權變換）。

3. 防禦性密封 (The Final Seal) - 強制性要求

在 calculate 函數的返回表達式中，必須強制執行以下三步驟清洗，否則不予驗收：

# 範例標準密封邏輯

final_expr = (
expr
.fill_nan(0) # 1. 殺死 NaN
.fill_null(0) # 2. 補齊空值
.clip(lower_bound=-5, upper_bound=5) # 3. 物理封頂，防止 SVD 爆炸
)

4. 實作流程規範 (Workflow for AI)

[辨識]：識別指標原始邏輯屬於 A, B, C, D 哪一類。

[硬化]：套用上述轉化公式，計算 \_pos, \_osc, \_mem。

[密封]：對所有輸出進行 fill_nan 與 clip 處置。

[重命名]：所有欄位必須以 feat\_ 為前綴。

[自檢]：確保 673 行數據下，特徵的標準差 (std) 絕對不能為 0。

5. 特徵命名契約

格式：feat*[IndicatorName]*[SubFeature]

範例：feat_VWAP_pos, feat_VWAP_osc, feat_SMC_ob_decay。
